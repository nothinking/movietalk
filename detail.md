## 1. 배경 및 목표

- **학습자 프로필**: 47세 한국인, 영어 공부 20년차
- **학습 목표**: 영어 영화를 보고 이해하고 싶음
- **프로젝트 목표**: 효과적인 공부 방법을 찾고, 학습을 돕는 앱을 직접 만들고 싶음

## 2. 영화 영어가 어려운 이유

- 슬랭 (slang)
- 축약 발음 (gonna, wanna, kinda 등)
- 빠른 대화 속도
- 문화적 맥락

## 3. 추천 공부 방법

### 3단계 학습법

| 단계 | 방법 | 설명 |
|------|------|------|
| 1단계 | 능동적 시청 (Active Watching) | 영어 자막으로 시청, 흐름 파악 후 모르는 표현 메모 |
| 2단계 | 장면 단위 반복 (Scene Mining) | 2~3분 장면 반복 청취, 받아쓰기(dictation) → 자막 비교 |
| 3단계 | 쉐도잉 (Shadowing) | 배우의 억양, 리듬, 연음을 따라 말하기 |

### 장르 선택 (난이도 순)

시트콤 (Friends, The Office) → 드라마 → 영화

## 4. 핵심 인사이트: "말할 수 없으면 들을 수 없다"

- 발음할 수 없는 소리는 인식하기 어려움 (언어학적으로도 검증된 원리)
- 한국인의 뇌가 영어 소리를 한국어 음운 체계로 매핑하려 하기 때문에 안 들림
- **해결책**: 실제 발음을 한글로 보여주어 인식의 갭을 줄임

### 한국인이 특히 안 들리는 핵심 패턴

#### 축약 & 연결 (Reduction & Linking)

| 실제 발음 | 교과서 영어 | 한글 근사 표기 |
|-----------|-------------|----------------|
| gonna | going to | 거나 |
| wanna | want to | 워나 |
| gotta | got to | 가라 |
| shoulda | should have | 슈다 |
| coulda | could have | 쿠다 |
| lemme | let me | 레미 |
| gimme | give me | 기미 |
| dunno | don't know | 더노 |
| kinda | kind of | 카인다 |
| outta | out of | 아우라 |

#### 탈락 (Sound Dropping)

- last night → "래스나잇" (t 탈락)
- just do it → "저스두잇" (t 탈락)
- center → "세너" (t가 ㄹ처럼)
- twenty → "트웨니" (t 탈락)

#### 연음 (Linking)

- check it out → "체키라웃"
- get out of → "게라우러"
- not at all → "나래럴"
- pick it up → "피키럽"
- turn it off → "터니로프"

#### 한국인이 구분 못하는 소리

| 구분 | 소리 A | 소리 B | 한글로는 |
|------|--------|--------|----------|
| R vs L | right / light | 혀 위치 다름 | 둘 다 "ㄹ" |
| F vs P | fan / pan | 윗니+아랫입술 vs 양 입술 | 둘 다 "ㅍ" |
| V vs B | vote / boat | 윗니+아랫입술 vs 양 입술 | 둘 다 "ㅂ" |
| TH vs D | they / day | 혀끝 내밀기 vs 안 내밀기 | 둘 다 "ㄷ" |
| Z vs J | zoo / jew | 치찰음 vs 파찰음 | 둘 다 "ㅈ" |

## 5. 앱 컨셉: MovieTalk

### 핵심 UX

> 유튜브 영상을 보다가 **일시정지하면** 해당 문장의 **실제 발음(한글) + 해석**이 표시됨

### 주요 기능

1. **영상 재생 + 영어 자막 표시**
2. **일시정지 시 학습 패널 표시**
   - 원문 (영어 대사)
   - 실제 발음 (한글 표기)
   - 한국어 해석
   - 발음 포인트 (축약/연음/탈락 설명)
3. **표현 저장 & 복습** — 모르는 표현 저장, 나중에 복습
4. **문장 반복 재생** — 특정 문장 구간 반복
5. **재생 속도 조절** — 0.5x ~ 1.25x
6. **문장 목록** — 전체 문장 리스트에서 원하는 문장 바로 선택

### 프로토타입

- React(JSX)로 프로토타입 완성
- 데모 데이터 8개 문장 포함
- 실제 앱에서는 YouTube IFrame API + 자막 파싱으로 연동

## 6. 실제 발음 추출 기술

### 방법 1: 자막 텍스트 → AI로 발음 생성 (현실적, MVP 추천)

```
유튜브 자막 (.srt) → Claude/GPT API → 한글 발음 + 해석 + 포인트
```

- `yt-dlp`으로 자막 추출
- LLM에 영어 문장을 보내면 축약/연음/탈락을 반영한 한글 발음 생성
- **장점**: 구현 간단, 품질 양호
- **단점**: 화자의 실제 발화 속도/억양 미반영

### 방법 2: 음성 분석 → 발음 추출 (정밀)

```
오디오 → Whisper (word-level timestamp)
       → phonemizer (IPA 변환)
       → IPA → 한글 매핑
```

- Whisper가 실제 발화를 인식 (gonna, wanna 등 축약형 자동 감지)
- IPA(국제음성기호)를 한글로 규칙 기반 변환
- **장점**: 실제 발화 반영
- **단점**: 파이프라인 복잡, 한글 매핑 품질 관리 필요

### 추천: 하이브리드 방식

```
유튜브 영상
    ↓
yt-dlp (자막 + 오디오 추출)
    ↓
┌──────────────────┬──────────────────┐
│ Whisper           │ 자막 텍스트        │
│ (실제 발화 인식)    │ (원문 대사)        │
└────────┬─────────┴────────┬─────────┘
         └──── 합쳐서 ──────┘
                  ↓
          Claude API에 전달
                  ↓
       한글 발음 + 해석 + 발음 포인트
```

**전략**: MVP는 방법 1로 빠르게 구현 → 이후 Whisper 추가로 정밀도 향상
